Alf — The development side

  • Alf is an open-source project implemented in Ruby
  • It is distributed under a MIT licence
  • The source code is hosted on Github
  • The issue tracker is hosted on Github as well

Contribute

The overal mechanism to contribute is by forking the project on Github. There are lots of stuff you can contribute to:

The library itself

  • You'll have to master Ruby, but it's a good opportunity to learn this wonderful language.
  • Everything is welcome except non-relational proposals (which can still be made, but will be kindly rejected for sure):
  • Fixing bugs; Implementing new options; New features; New rubyisms; Contributing to experimental operators; and so on.

The documentation

  • Alf documentation is maintained in text files (using the markdown format) and hosted on Github.
  • Commands and operator descriptions may be found in doc/commands and doc/operators, respectively.
  • The pages of this web site can be found in doc/gh-pages and its subfolders.

Ideas, issues and critical reading

  • New ideas are welcome, for almost everything. Implemented ideas are prefered, but not required.
  • Critical reading from TTM-aware people is necessary and welcome. Create an issue in our bug tracker if your review/question/proposal is limited to Alf (misunderstanding of a Tutorial D operator by Alf, for example). Start a new thread on the TTM mailing list if it is more general (like discussing an experimental operator implemented in Alf).
